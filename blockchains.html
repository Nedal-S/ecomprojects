How Bitcoin Mining Works

Mining is the process by which new blocks are added to the Bitcoin blockchain and new bitcoins enter circulation. Miners compete to solve a cryptographic puzzle based on SHA‑256 hashing. The first miner to find a hash below the current network target broadcasts the block to the network and receives the block reward plus transaction fees.

Step‑By‑Step

  1. Transaction Pool: Pending transactions gather in the mempool.
  2. Block Assembly: Each miner constructs a candidate block of transactions.
  3. Hashing Race: Miners iterate a “nonce” field, hashing the block header billions of times per second.
  4. Proof‑of‑Work: When a hash meets the difficulty target, the miner broadcasts the block.
  5. Consensus: Other nodes validate the block. Once accepted, it becomes part of the immutable ledger.

Difficulty & Halving

The network difficulty adjusts every 2 016 blocks to keep block time ~10 minutes. Approximately every four years, the block reward halves—making mining progressively harder and reinforcing Bitcoin’s supply schedule.

Hardware Evolution

  • CPU → GPU → FPGA → ASIC hardware optimized for SHA‑256.
  • Modern ASICs perform >100 TH/s while consuming less than 30 J/TH.

Fun fact: The combined SHA‑256 hash‑rate now exceeds 500 EH/s—an astronomical 5 × 1020 hashes per second.
